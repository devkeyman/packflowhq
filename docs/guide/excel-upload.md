# 엑셀 업로드 가이드 (작업 지시서 일괄 등록)

## 개요

생산 관리 페이지에서 엑셀 파일을 이용하여 작업 지시서를 일괄 등록할 수 있습니다.

## 엑셀 템플릿

### 다운로드 방법

1. 생산 관리 페이지(`/production`)에서 **[엑셀 업로드]** 버튼 클릭
2. 다이얼로그에서 **[템플릿 다운로드]** 버튼 클릭
3. `작업지시서_업로드_템플릿.xlsx` 파일이 다운로드됩니다

### 컬럼별 입력 규칙

| 컬럼명 | 필수 | 타입 | 허용값 | 비고 |
|--------|------|------|--------|------|
| 공급구분 | O | 텍스트 | `원청`, `하청` | |
| 업체명 | O | 텍스트 | 자유 입력 | |
| 제품명 | O | 텍스트 | 자유 입력 | |
| 제품코드 | | 텍스트 | 자유 입력 | |
| 수량 | O | 숫자 | 1 이상 정수 | |
| 단위 | | 텍스트 | `EA`, `BOX`, `SET`, `KG`, `M` | 미입력 시 기본값 없음 |
| 마감일 | O | 날짜 | `YYYY-MM-DD` 형식 | 예: 2026-12-31 |
| 우선순위 | | 텍스트 | `낮음`, `보통`, `높음`, `긴급` | 미입력 시 `보통` |
| 작업유형 | | 텍스트 | `생산`, `조립`, `포장`, `검사`, `기타` | |
| 작업지시사항 | | 텍스트 | 자유 입력 | |

## 업로드 절차

### 1. 파일 선택

- 생산 관리 페이지에서 **[엑셀 업로드]** 클릭
- 파일 선택 영역을 클릭하여 `.xlsx` 또는 `.xls` 파일 선택
- 파일을 선택하면 자동으로 파싱되어 미리보기로 이동

### 2. 미리보기 확인

- 파싱된 데이터가 테이블 형태로 표시됩니다
- 각 행의 유효성 검사 결과 확인:
  - 초록색 체크: 유효한 데이터
  - 빨간색 X: 오류가 있는 데이터 (오류 필드가 빨간색으로 표시)
- 오류가 있는 행은 업로드에서 자동 제외됩니다
- **[업로드]** 버튼에 유효 건수가 표시됩니다

### 3. 업로드

- **[N건 업로드]** 버튼 클릭
- 서버에서 건별로 처리하여 결과를 반환합니다
- 성공/실패 건수와 실패 사유가 표시됩니다
- 성공한 건은 즉시 목록에 반영됩니다

## 오류 대응

### 프론트엔드 유효성 검사 오류 (미리보기 단계)

| 오류 | 원인 | 해결 |
|------|------|------|
| "필수 항목입니다" | 필수 컬럼 값이 비어있음 | 해당 셀에 값 입력 |
| "허용값: 원청, 하청" | 공급구분 값이 잘못됨 | `원청` 또는 `하청`으로 수정 |
| "1 이상의 숫자를 입력하세요" | 수량이 0이거나 숫자가 아님 | 1 이상의 정수 입력 |
| "YYYY-MM-DD 형식으로 입력하세요" | 날짜 형식 오류 | 날짜 형식을 `2026-12-31`로 수정 |

### 서버 처리 오류 (업로드 결과 단계)

| 오류 | 원인 | 해결 |
|------|------|------|
| "마감일은 오늘 이후여야 합니다" | 과거 날짜 입력 | 오늘 이후 날짜로 수정 |
| 기타 서버 오류 | 서버 내부 오류 | 해당 건만 별도 등록 |

### 오류가 있는 행 수정 방법

1. 다이얼로그를 닫고 엑셀 파일에서 오류 데이터를 수정
2. 다시 **[엑셀 업로드]** 버튼을 클릭하여 수정된 파일 업로드

## API 정보

- **Endpoint**: `POST /api/work-orders/bulk`
- **Request Body**: `CreateWorkOrderRequest[]` (JSON 배열)
- **Response**: `BulkCreateResponse` (전체/성공/실패 건수 + 건별 결과)
